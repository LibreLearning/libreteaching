\section{Licencias libres}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{El marco legal para licencias libres}
\begin{itemize}
\item Las licencias libres se basan en las leyes de copyright y proveen al usuario de ciertas libertades. Estas libertades se obtienen como permisos que \alert{no pueden ser ejercitados} sin la licencia (por defecto, ``all rights are reserved'').
\item \alert{Hacking legal:} las licencias libres se comportan como cualquier otra licencia excepto en que ceden una serie de derechos al usuario en vez de restringirlos.

\end{itemize}

\alert{Legalmente, la única diferencia entre el software privativo y el software libre está en la licencia (en los términos de uso). Las licencias, ya sean libres o no, se basan en las leyes de \textit{copyright} de cada país.}

\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{Ejemplo}

Implementar una licencia libre básica es muy sencillo:

\begin{block}{Free License Example}
Copyright (c) 2008 Foobar Developers. All rights reserved. 

\medskip
Redistribution and use in source and binary forms, with or without modification, are permitted provided that the redistributions of source code must retain the above copyright notice.
\end{block}

\textit{¡Eso es todo!}

\end{frame}

\begin{frame}
\frametitle{La Definición de Software Libre}
El software libre se basa en licencias libres. Son el mecanismo legal para implementar las cuatro libertades. Cuando recibes el software tienes:

\bigskip

\begin{itemize}
\item {\alert{Libertad 0} La libertad de usar (ejecutar) el programa, para cualquier propósito.}
\item {\alert{Libertad 1} La libertad para estudiar cómo funciona el programa y adaptarlo a sus necesidades.}
\item {\alert{Libertad 2} La libertad de redistribuir copias.}
\item {\alert{Libertad 3} La libertad de mejorar el programa y de publicar tus mejoras al público de manera que toda la comunidad se pueda beneficiar.}
\end{itemize}

\pause
\small
Las libertades 1 y 3 requieren acceso al código fuente. 
%Todas las libertades han de ser ``usables'' \alert{a la vez}. 

 \pause
 \begin{center}
 \alert{Software libre $\neq$ software gratis}
 \end{center}
\normalsize
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{Algunos conceptos relacionados}

\begin{itemize}
\item \alert{Uso:} El derecho a usar (ejecutar) el programa, para cualquier propósito.
\item \alert{Redistribución:} El acto de copiar un programa y dárselo a otras personas.
\item \alert{Obra derivada:} Un programa basado en otro programa, reutilizando su código fuente.
\item \alert{Atribución de autoría:} La obligación de reconocer la autoría de una obra cuando se efectúe cualquier cambio, tal como crear obras derivadas o redistribuirla.
\end{itemize}

\small
La titularidad de un programa no cambia: el programa siempre tiene un titular de los derechos. 
\normalsize

\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{Licencias: conceptos}

License:
\begin{itemize}
\item Un ``contrato'' unilateral entre el titular/autor y el usuario.
\item Concede algunos derechos a los usuarios de la obra publicada.
\item En el software privativo, si no lo aceptas, no puedes usarlo. 
\item En el software libre las EULAs no son necesarias.

\end{itemize}


\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{``Piratería''}

A veces, las copias no autorizadas reciben el nombre de:
\begin{center}
\large{``Piratería''}
\end{center}

Hay organizaciones que luchan contra la ``piratería''.
Ejemplo:
\begin{itemize}
\item Business Software Alliance (BSA).
\end{itemize}

Se trata de un término controvertido porque significa robar barcos en alta mar (con violencia). 

\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{Tipos de licencias libres}

Cada \alert{licencia libre}, no importa qué tipo de obra, tiene que garantizar  \alert{las cuatro libertades} mencionadas anteriormente para el caso de software (uso, copia, modificación y redistribución).\\\pause

\medskip

Hay licencias de software más permisivas y otras menos (las licencias menos laxas se llaman ``copyleft''). \\\pause

\medskip
\small
Nótese que dos licencias libres no compatibles no significa que una sea ``menos libre'' que la otra.
\normalsize
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{Tipos de licencias libres (2)}
Las licencias libres pueden clasificarse en dos categorías:\\\pause
\begin{itemize}
\item \alert{Licencias copyleft:} El autor retiene el copyright y permite la redistribución y modificación siempre y cuando la redistribución sea bajo las mismas condiciones (bajo la misma licencia). Añadidos y modificaciones por parte de otros usuarios han de ser también licenciadas bajo la misma licencia ``copyleft''. También se conocen como ``licencias recíprocas'' o ``share-alike''.

\item \alert{Licencias permisivas:} El autor retiene el copyright sólo para excluir garantía y requiere atribución en obras modificadas, pero permite la redistribución y modificación en cualquier obra, incluso de manera propietaria.\\\pause
\end{itemize}

Nótese que ambos tipos son licencias de sofware libre. Con el primer tipo no se pueden crear obras derivadas propietarias.

\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{¿Qué es el copyleft?}
\begin{itemize}
    \item El copyleft posibilita que todos los usuarios que reciben una copia del programa también pueden hacer uso de las cuatro libertades.
    \item Es una defensa activa de las libertades de los usuarios
    \item La \alert{cláusula copyleft} puede tener diferentes formas de implementación, pero todas comparten un mismo concepto: \alert{la distribución de cualquier versión de este programa ha de ser bajo la misma licencia.}

\end{itemize}

\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{Licencia dual}

El titular de los derechos puede distribuir su trabajo bajo más de
una licencia. Los motivos pueden ser:

\begin{itemize}
\item Compatibilidad entre licencias (Perl, Mozilla/Firefox, MySQL).
\item Segregación de mercado para diferentes modelos de negocio (MySQL Enterprise)
\item Permite al titular ofrecer adaptaciones, publicaciones rápidas, generar otras versiones derivadas o ceder algunos derechos a terceras partes (p.ej. redistribuir versiones propietarias).
\end{itemize}

El autor o el titular de los derechos puede siempre elegir la licencia de su programa; por ejemplo, puede cambiar de una licencia libre a una licencia propietaria. Sin embargo, las versiones con licencias libres siempre existirán y las condiciones no cambiarán (p.ej. ``forks'' como en GForge).
                              
\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{Conclusiones}

Resulta esencial poner atención a las licencias libres y aprender a diferenciar entre ellas, no sólo para determinar si estamos sujetos a alguna restricción --como usuarios-- o de lo que estamos ofreciendo a los usuarios --como autores--, sino también para entender posibles incompatibilidades derivadas de combinar contenidos bajo licencias libres diferentes.

\end{frame}

\begin{frame}
\frametitle{Tipos de licencias de software}

\begin{itemize}
\item {\alert{Licencias privativas:} todos los derechos reservados. Se paga por cada licencia. Ejemplo: Licencia de Microsoft Windows o de Mac OSX de Apple.}

\item {\alert{Licencias libres:} libertad de copiar, modificar y redistribuir. Hay dos grandes variantes (culturas):}

\begin{itemize}
\item {Licencias permisivas (no copyleft): no hay restricciones (BSD, Apache, X11)}
\item {Licencias copyleft: cualquier modificación mantiene todas las libertades originales (GPL).}
\end{itemize}
\end{itemize}
\medskip

Fuera del software libre, el concepto es más laxo: CC (licencias a la carta).

\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{Restricciones admisibles}
\begin{itemize}
    \item Reconocimiento/atribución (la atribución no debe impedir el uso normal de la obra).         
    \item Transmisión de libertades (como el copyleft o el ``share-alike'').
    \item Protección de libertades (acceder al código fuente o prohibición de me ``medidas técnicas'' como el DRM). 
\end{itemize}

\end{frame}

\begin{frame}
\frametitle{La Licencia GNU GPL}

\begin{center}
\item GNU GPL = GNU General Public License.
\end{center}

Conceptos de la GPL:
\begin{itemize}
\item Creada por la Free Software Foundation para el proyecto GNU.
\item Sin embargo, es muy usada en software no GNU.
\item Probablemente la licencia de software libre más popular: alrededor del 70\% de los proyectos en FreshMeat están bajo la GPL.
\item Software popular bajo la GPL: Linux, GNOME, Emacs, GCC, Epiphany\ldots
\end{itemize}

\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{La Licencia GNU GPL}
¿Qué hace a la GNU GPL tan especial?
\pause
\begin{itemize}
\item Fue la primera licencia con la cláusula ``copyleft''.
\item Todas las licencias copyleft se han basado en la GPL, incluidas las de Creative Commons y la de Wikipedia.
\item Sin la GPL, el copyleft sería simplemente una idea, y la Cultura Libre no existiría (al menos, tal y como la conocemos).
\end{itemize}

\pause
\alert{Las licencias libres son el fundamento jurídico-filosófica del software libre}

\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{La Licencia GNU GPL}

Características de la GPL
\begin{itemize}
\item Garantiza las cuatro libertades del software libre.
\item Está pensada para garantizar siempre y a todo el mundo estas cuatro libertades (``copyleft'').
\item Como en EE.UU. las patentes son admisibles, la GPL incluye mecanismos que evitan que estas invaliden algunos de los derechos de la licencia.
\item Tiene tres versiones.
\item Código bajo GPL no puede ser redistribuido mezclado con código de una licencia incompatible.
\end{itemize}

\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{La Licencia GNU GPL}

\begin{itemize}
\item Basada en la licencia de Emacs, la primera licencia copyleft (1986).
\item \textit{GPL versión 1} (1989). Genérica, independiente del programa, ``version 1 or later''.
\item \textit{GPL versión 2} (1991). Cláusula ``Liberty or dead'' (pensada como protección contra patentes).
\item \textit{GPL versión 3} (2007). ``Tivoización'', patentes y DRMs. Elaborada después de discutirla en comunidad.
\end{itemize}

\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{La GPL, versión 2 (GPLv2)}

\begin{itemize}
    \item Escrita por Richard Stallman y publicada por la FSF en 1991
    \item La licencia más popular de software libre: cubre el 50-70\% de todo el software libre.
    \item Es más que una licencia de software: es un contrato social.

\end{itemize}

\pause

{\centerline{\large \alert{¿Por qué cambiarla?}}}

\pause

Tras más de 15 años, hacía falta actualizarla para que siguiese siendo eficaz ante los desafíos tecnológicos del siglo XXI.

\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%


\begin{frame}
\frametitle{Cambios en la GPLv3}
\begin{itemize}
\item {Cambios sustanciales}
\begin{itemize}
\item {Gestión de Restricciones Digitales (DRM)} 
\item {Mayor protección contra la amenaza de patentes.}
\item Leyes WIPO anti-elusión (\textit{anti-circumvention}) que prohíben software libre (DMCA y EUCD)
\item Clarifica la compatibilidad de la licencia (permisos adicionales)
\end{itemize}

\item {Cambios menores}
\begin{itemize}
\item Adaptación a los avances tecnológicos 
\item Clarificaciones para facilitar su uso y comprensión
\item Internacionalización (\textit{convey/distribution})
\end{itemize}
\end{itemize}

{\centerline{Muchos cambios pero los principios fundamentales no cambian.}}

\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%


\begin{frame}
\frametitle{Gestión de Derechos Digitales (DRM)}

\begin{itemize}
\item El DRM es incompatible con las libertades del usuario (es tratado como ``enemigo'').
\item Pérdida de control sobre lo que hace el software.
\item \textit{Tivoización:} restricciones al usuario para poder actualizar o modificar el software incrustado que deje la GPLv2 en papel mojado.
\item Legislación global (OMPI) que impide la autodefensa.

\end{itemize}

\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{Gestión de Derechos Digitales (DRM)}

Qué hace la GPLv3:

\begin{itemize}
\item Neutraliza las leyes que prohíben el software libre (DMCA, EUCD)
\item NO prohíbe implementar DRM con software GPL.
\item Pero se puede escribir software interoperable con él.
\item Neutraliza la tivoización: se debe proveer la forma en que el usuario ejerca su derecho a la modificación del software GPL integrado.
\end{itemize}

\alert{La GPL no restringe lo que las personas pueden hacer con el software; solo se les impide que traten de restringir la libertad de otros.}

\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{Patentes de software}
La defensa contra las patentes de software ya se implementó en la GPLv2 con la cláusula "<Libertad o muerte">.

\begin{itemize}
\item Si el software bajo GPL incluye patentes que imponen condiciones, no puede ser distribuido.
\item Evita software libre zombie: software que sería libre si no hubiese patentes.
\end{itemize}

{\centerline{Esta cláusula se mantiene en la GPLv3.}}


\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{Patentes de software}
La GPLv3 añade una protección mayor ante la amenaza de las patentes mediante ingeniería jurídica:

\begin{itemize}
\item Quien distribuya software GPL, debe proveer de los eventuales derechos de patentes para poder ejercer las libertades que la GPL le otorga.
\item Si cualquier usuario pretende ejercer un derecho de patentes contra otro mediante la GPL, su licencia de uso se termina.
\item Usuarios y desarrolladores pueden trabajar con software bajo GPLv3 sin preocuparse de que un colaborador intente demandarle más tarde por infracción de patentes.

\end{itemize}


\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{Compatibilidad de licencias}

Compatibilidad = posibilidad de combinar código de licencias distintas.

\begin{itemize}
\item GPLv3 incrementa las posibilidades compatibilidad con otras licencias libres.
\item ¿Cómo lo hace? Permite requerimientos adicionales:
\begin{itemize}
\item Responsabilidad: permite añadir notas de garantía o de responsabilidad.
\item Limitaciones publicitarias: restricciones sobre el uso de nombres/marcas/instituciones.

\end{itemize}
\end{itemize}

La GPLv3 es más modular, más compatible, permite experimentos y será compatible con futuras licencias copyleft.

\end{frame}



%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\frametitle{Proceso de elaboración de la GPLv3}

Proceso de consulta pública:
\begin{itemize}
\item Se prolongó durante año y medio (entre el 16 de enero de 2006 del primer borrador y el 29 de junio de la versión oficial).
\item Cuatro borradores. 
\item Cinco conferencias internacionales (Boston, Porto Alegre, Barcelona, Tokio y Bruselas)
\end{itemize}

El cambio mayor respecto a versiones anteriores ha sido la forma de reelaboración de la licencia, discutida y consensuada por la comunidad.
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%


\begin{frame}
\frametitle{Proceso de elaboración de la GPLv3}

Algunos puntos donde hubo más discusión:
\begin{itemize}
\item Simplicidad y longitud
\item Por qué la licencia GPLv3 dice que los permisos adicionales son eliminables
\item Cómo aborda la licencia GPLv3 la EUCD y la DMCA
\item La GPLv3 y los dispositivos integrados
\item Cómo trata la licencia GPLv3 la proliferación de licencias
\item Tivoización: implementación y daños
\end{itemize}

\end{frame}

\begin{frame}
\frametitle{Licencia GNU LGPL}

\begin{itemize}
\item LGPL empezó como ``Library GPL''. Fue renombrada después a ``Lesser GPL''.
\item LGPL mantiene todas las libertades y restricciones sobre el software licenciado con GPL, con una única excepción:
\begin{itemize}
\item Software bajo la LGPL puede ser integrado/enlazado con cualquier otro software, sin limitaciones (incluyendo software privativo).
\end{itemize}
\end{itemize}

¿Por qué la LGPL?
\begin{itemize}
\item Creada para promocionar el uso de bibliotecas GPL en cualquier otro software (p.ej., GNU libc).
\item Más tarde, la FSF comprobó que la LGPL no ayudaba en la creación de nuevo software libre, así que decidieron renombrarla a ``lesser'' y desaconsejar su uso generalizado.
\end{itemize}

\end{frame}

\begin{frame}
\frametitle{Otra licencia robusta: Affero GPL}

La Affero GPL (AGPL) es una licencia de software libre, del tipo copyleft, publicada por la Free Software Foundation.

\begin{itemize}
\item Derivada de la GNU GPL
\item Contiene una cláusula que requiere la distribución de cualquier código fuente modificado que se esté ejecutando en una red de ordenadores.
\item Tiene como objetivo el caso de un software GPL modificado que no se distribuye porque la licencia GPL no requiere que sea así (p.ej. servicios web o aplicaciones on-line).
\end{itemize}
\end{frame}

\begin{itemize}

\end{itemize}

\begin{frame}
\frametitle{Licencias BSD}

\begin{itemize}
\item Basadas en la licencia BSD original (fines de años 80).
\item Muy populares: X-Window, Tcl/Tk, etc.
\item Puedes redistribuir la obra de cualquier manera (fuente o binario) 
pero con todas las notas de copyright (atribución al autor original)
\item Cláusula de carencia de garantía. Muy criticada, esta cláusula es para evitar problemas legales. Es una cláusula común también en el software privativo.
\end{itemize}

\end{frame}

%
{\small
\begin{itemize}
%
\item {\bf Licencia X-Window (X11)}. Se aplica al subsistema gráfico más usado en el mundo Unix. Es muy similar a la BSD y se la conoce también como ``MIT License''.
\item {\bf Zope Public License 2.0}. Usada en el servidor de aplicaciones Zope y productos relacionados. Similar a la BSD. 
\item {\bf Apache}. Muy usada, en módulos de Apache realizados por terceros. 
%
\item {\bf Dominio público}.No es propiamente una licencia, sino el estado en el que una obra queda una vez expiran los derechos de copyright. En términos prácticos es muy similar a la BSD.
\end{itemize}
}
%
\end{frame}
